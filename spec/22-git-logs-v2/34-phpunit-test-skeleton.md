## Why PHPUnit (recap from §32)

- Native PHP test runner, ships with Composer; standard for WordPress plugin ecosystems.
- Asserts pure-PHP behavior **without booting WordPress** — forces the plugin code to be WP-decoupled at the unit layer (mappers, validators, parsers, signing-string builder).
- Fast: 1000+ tests in <10 s on a laptop. Bats covers the slower CLI surface separately.

## Bootstrap rules (`bootstrap.php`)

The bootstrap MUST do exactly this, in order:

1. `require __DIR__ . '/../../vendor/autoload.php';`
2. Define constant `GL_TEST_MODE = true` so plugin code can branch on test-only paths (e.g., skip WP-only function calls).
3. Register a PSR-4 autoloader for `inc/` namespace; **do NOT load WordPress core**.
4. Provide minimal WP function shims required by the units under test (`__()`, `esc_html()`, `wp_json_encode()`, `current_time()`). Shims live in `tests/phpunit/wp-shims.php` and are explicit about what they fake.
5. Set the default timezone to UTC. Tests MUST NOT depend on the host timezone.

The bootstrap MUST NOT:
- Open a SQLite connection (each test owns its own).
- Run migrations or seed data (each test class controls that via `BaseDbTestCase`).
- Read from `$_ENV` for anything other than `BATS_GL_NAMESPACE` and a fixture-path override.

## Base classes

### `BaseTestCase`

- Extends `PHPUnit\Framework\TestCase`.
- Provides `assertJsonStructure(array $expected, string $json)` (deep key existence + type check).
- Provides `assertContainsAuditCode(string $code, string $haystack)` for response-envelope checks.
- No DB access. Use this for pure functions (mappers, parsers, builders).

### `BaseDbTestCase` (extends `BaseTestCase`)

- `setUp()` opens a fresh `':memory:'` SQLite, calls the production `Activator::run()` to create schema + seeds, then exposes `$this->db`.
- `tearDown()` closes the DB (no persistence between tests).
- Provides `seedRow(string $table, array $cols): int` (INSERT helper, returns lastInsertRowId).
- Provides `auditRows(?string $action = null): array` (SELECT helper for assertions).
- Used by Migrations/, Ssh/, Config/, Validators/ (anything touching the DB).

> Both base classes MUST be `abstract`. Tests cannot instantiate them directly.

## Coverage targets (build-fails-on-shortfall)

| Class under test | Required coverage | Required test count |
|------------------|-------------------|---------------------|
| `RepoUrlParser` | 100% line + branch | ≥12 (one per URL shape in `repo-url-cases.json`) |
| `TempTokenValidator` | 100% branch | one test per validation step (1–8) per §05 |
| `SshSignatureValidator` | 100% branch | one test per validation step (1–10) per §31 |
| `CanonicalStringBuilder` | 100% line | byte-equality against every fixture in `canonical-strings/` |
| `ErrorResponder` | 100% line | one test per row in `error-code-map.json` (mirrors §15) |
| `Activator` | ≥90% line | idempotent-on-second-run + MigrationState assertion |
| `SeedLoader` | ≥90% line | every lookup table seeded; admin edits preserved |
| `Redactor` | 100% branch | one test per redacted field (TempToken, Token, Fingerprint, Signature) |

A `tests/phpunit/coverage-check.php` script runs after `phpunit --coverage-clover` and fails Stage 4 if any row above is short. Drift between §15 and `error-code-map.json` MUST also fail Stage 4.

## File template

```php
<?php
declare(strict_types=1);

namespace GitLogs\Tests\Auth;

use GitLogs\Tests\BaseDbTestCase;
use GitLogs\Auth\SshSignatureValidator;

final class SshSignatureValidatorTest extends BaseDbTestCase
{
    private SshSignatureValidator $sut;

    protected function setUp(): void
    {
        parent::setUp();
        $this->sut = new SshSignatureValidator($this->db);
    }

    public function testRejectsWhenAuthModeHeaderMissing(): void
    {
        // arrange
        // act
        // assert
    }
}
```

Rules:
- `declare(strict_types=1);` mandatory.
- One assertion per test (`@dataProvider` for table-driven cases; each row is its own implicit test).
- Test method names: `test<Behavior>When<Condition>` — present tense, no underscores.
- No `sleep()`. Use injected clocks (`Clock` interface) so timestamp tests are deterministic.
- No network. No filesystem writes outside `BATS_TMPDIR`-equivalent.

## Fixture file rules

- `canonical-strings/` MUST be a **symlink** to `../bats/fixtures/canonical-strings`. Both test layers compare against the same bytes; otherwise drift goes undetected.
- `error-code-map.json` is regenerated by a script that parses §15. CI Stage 6 (Verify) re-runs the script and `git diff --exit-code` — drift fails the build.
- `ssh/id_ed25519{,.pub}` is a **committed** ephemeral test keypair. Never reused for any non-test purpose; documented in the file header. Regenerated only via `tests/phpunit/regen-fixtures.sh`.

## phpunit.xml.dist contract

Required elements (no implementation — just the contract):

- `<testsuite name="plugin">` includes `tests/phpunit/`.
- `<coverage>` includes `inc/`, excludes `inc/Admin/Views/` (templates, no logic).
- `<php>` block sets `date.timezone=UTC` and `error_reporting=E_ALL`.
- Strict mode flags ON: `failOnRisky`, `failOnWarning`, `failOnIncomplete`, `failOnDeprecation`.
- `cacheDirectory=".phpunit.cache"` — never committed (in `.gitignore`).

## Static analysis gate (Stage 3 hand-off)

Although static analysis is Stage 3 (Build), it gates Stage 4 (Unit). PHPUnit MUST NOT run if `phpstan --level=max` failed. The `ci.yml` (§35) wires this dependency.

PHPStan baseline rules:
- `inc/` MUST be `level=max` clean.
- `tests/phpunit/` MUST be `level=8` clean (one notch lower because reflection in test fixtures is allowed).
- No `// phpstan-ignore` comments in `inc/` without a linked issue number.

### §34.D — Coverage-gap detection contract (Normative)

A CI step (slot-TBD or `linter-scripts/check-test-coverage-gap.sh`-class) MUST grep both files for **every T-ID** declared in §97 AC-05 + AC-12 matrices. Missing IDs MUST raise `GL-TEST-COVERAGE-GAP` per §15. Renaming a method without updating the T-ID comment is a SPEC VIOLATION (the comment IS the binding). Adding a new T-row in §97 without a matching method/case in §34.A or §34.B in the SAME PR is a lockstep violation per Lesson #36 (link-don't-restate also implies link-must-exist).

### §34.E — Clock-injection contract reference (Normative)

Both scaffolds depend on `GitLogs\Clock\ClockInterface` per §97 AC-29 §5 (`now(): int`, `nowMs(): int`). Production: `SystemClock`. Tests: `FakeClock` (`advance(int $seconds)`, `set(int $unix)`). Direct `time()/microtime()/date()/DateTimeImmutable("now")/gmdate()/hrtime()` calls inside `inc/Logger/`, `inc/Ingest/`, `inc/Auth/Ssh/`, `inc/RateLimit/` are FORBIDDEN — see §97 AC-29 §5 for the complete contract; restating the interface signature here is FORBIDDEN per Lesson #36.
