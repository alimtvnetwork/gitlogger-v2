### AC-89 — Server-side `GET /wp-json/git-logs/v3/autofix/{repoIdentityHash}/{frameId}` fix-bundle producer (K-series tier-1; Lane B reads only; single-consumption; Lesson #19 + Lesson #21 + Lesson #36)  `[critical]`

- **Then** the auditor MUST find the following five normative sub-contracts pinned in `47-server-autofix-endpoint.md` (delegated by name from this AC, not restated here per Lesson #21 discipline):

  1. **Endpoint binding (§2)** — `GET /wp-json/git-logs/v3/autofix/{repoIdentityHash}/{frameId}` on the v3 base per K1; `{repoIdentityHash}` 32-hex per §40 §3; `{frameId}` UUIDv4 per §43 §3; **Lane B only** for reads (Lane A → 401 `GL-AUTH-LANE-MISMATCH`); `Accept: application/vnd.riseup.git-logs.fix+json; v=1` REQUIRED (else 406); `Idempotency-Key` FORBIDDEN on GET (else 400 `GL-FIX-IDEMPOTENCY-FORBIDDEN`); `If-None-Match` OPTIONAL (304 on match — does NOT count as consumption); response `Content-Type: application/vnd.riseup.git-logs.fix+json; v=1`; strong `ETag` REQUIRED (`sha256(bundleId+targetHead+canonical-bytes)[0:16]` double-quoted); `Cache-Control: private, no-store` REQUIRED (intermediary caching FORBIDDEN — bundles are single-consumption secrets); 4 MiB compressed / 16 MiB decompressed body cap (mirrors §44 §2). FORBIDDEN: any HTTP method other than GET → 405; any `X-GL-Auth-Mode` other than `ssh`; serving from any base other than `/wp-json/git-logs/v3/`; honouring `Idempotency-Key` on this GET; setting `Cache-Control: public`; responding without an `ETag`.
  2. **11-step strict server validation + lookup order (§3)** — (1) method/path exact match → (2) path-param shape (32-hex + UUIDv4) → (3) `Accept` exact match → (4) `Idempotency-Key` ABSENT → (5) Lane B 12-step per §31 link-don't-restate → (6) `(repoIdentityHash)` resolves to a `Repo` visible to authenticated `Profile` (else 404 — NOT 403, to avoid existence-leak) → (7) `(repoIdentityHash, frameId)` row exists in `FixBundle` (else 404) → (8) `expiresAt > now()` (else 410 `GL-FIX-EXPIRED`) → (9) `consumedAt IS NULL` (else 410 `GL-FIX-CONSUMED` — single-consumption invariant per §44 §10) → (10) `If-None-Match` mismatch (match → 304, no body) → (11) token-bucket rate-limit per `OwnedByProfileId`. **Reordering FORBIDDEN.** Auth (5) MUST precede ownership (6) — unauthenticated callers cannot probe existence. Ownership (6) MUST precede bundle existence (7) — cross-tenant probes return identical 404 regardless of bundle presence. Expiry (8) MUST precede consume (9) — expired-but-already-consumed bundles return `GL-FIX-EXPIRED` (more actionable per §44 §9).
  3. **Bundle envelope production (§4)** — produce the 7-field closed-set envelope per §44 §3 (`bundleId` · `batchId` · `repoIdentityHash` · `targetHead` · `schemaBump` · `patches[]` · `expiresAt`) sourced from the `FixBundle` row; canonical lexicographic key order REQUIRED (deviation = `GL-SCHEMA-DRIFT`); `patches[]` shape `{path, mode, beforeSha256, afterSha256, unifiedDiff}` `additionalProperties: false`; full 17-entry forbidden field list OWNED by §44 §3 (NOT restated here). FORBIDDEN: any field outside the 7-field set; mutating `patches[].unifiedDiff` between bundle creation and download (bundles are immutable once written; re-derivation = new `bundleId`); non-canonical key order; emitting NDJSON.
  4. **Single-consumption + expiry semantics (§5)** — on successful 200, server MUST atomically (per spec/13 AC-22 link-don't-restate, single SQLite transaction with `BEGIN IMMEDIATE`): set `FixBundle.consumedAt = now()` + set `FixBundle.consumedByProfileId` + append `SystemEvent` row type `FixBundleConsumed` per §20. Partial writes = `GL-SCHEMA-DRIFT` and bundle MUST be considered consumed (do NOT retry — client already received body). Default TTL `FixBundleDefaultTtlSeconds` ConfigKv (default 86400 = 24h). Bundles past `expiresAt` → 410 (do NOT auto-extend, do NOT regenerate inline — client falls back to fresh upload per §44 §5 step 1). Concurrent-GET race: implementation MUST use `SELECT ... FOR UPDATE` or SQLite `BEGIN IMMEDIATE` + `WHERE consumedAt IS NULL` UPDATE-and-check-rowcount; application-level mutex / advisory lock FORBIDDEN (does not survive server restart).
  5. **Forbidden patterns (§6)** — Lane A for reads; client-constructed URLs bypassing server-issued `autofixDownloadUrl` from §43 §6 (→ 403 `GL-FIX-URL-CONSTRUCTED`); restating §44 §3 7-field envelope or 17-entry forbidden list inline; restating §44 §4 4-state outcome set inline (outcome is computed CLIENT-side after applying — server has no opinion); restating §31 12-step inline; restating §43 frame envelope or idempotency-key contract inline; restating `GL-FIX-*` / `GL-AUTH-*` semantics inline (cite by code name only — owned by §15); honouring `Idempotency-Key` on this GET; setting `Cache-Control: public` or omitting it; auto-regenerating expired bundle inline; coalescing v2 + v3 paths into a single handler; servicing partial-bundle resumption (byte-range → 416 `GL-FIX-RANGE-UNSUPPORTED`).

**AND** the K-series sibling slots §48/§49 ship in K3/K4 under AC-90/AC-91 (reserved); each MUST add a row to AC-80 Sibling File Delegation Map in the same phase as the file edit (Lesson #21 invariant);

  1. **Endpoint binding (§48 §2)** — `POST /wp-json/git-logs/v3/refresh-pointer` on the v3 base per K1; **Lane B only** for writes (Lane A → 401 `GL-AUTH-LANE-MISMATCH`); `Content-Type: application/json` REQUIRED (NDJSON / multipart → 415 `GL-PTR-CONTENT-TYPE-UNSUPPORTED`); `Idempotency-Key` OPTIONAL (5-min cache keyed on `(profileId, repoIdentityHash, currentHead, key)`; sha256-mismatched replay → 409 `GL-PTR-IDEMPOTENCY-CONFLICT`); response `Content-Type: application/json` (single §41 payload — same shape as on-disk `.riseup-git-logs.json`); response `Cache-Control: private, no-store` REQUIRED (per-profile rotated token in `uploadUrl`); 4 KiB body cap (>4 KiB → 413 `GL-PTR-BODY-TOO-LARGE` — guards smuggled-frames-in-refresh-pointer exploits); 10s server timeout; token-bucket rate-limit per `(profileId, repoIdentityHash)` capacity 30 refill 1/s (intentionally lower than §46 — refresh is bursty-rare).
