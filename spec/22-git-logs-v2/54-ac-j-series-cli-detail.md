### AC-86 — `riseup-git-logs` CLI auto-fix protocol (J-series tier-1; Lesson #19 + Lesson #21 + Lesson #36)  `[critical]`

- **Given** a successful upload per AC-85 §6 returns `autofixDownloadUrl` (https or null), and when non-null the CLI MAY download a server-prepared fix bundle and apply it to the working tree under a hybrid 4-state propose-diff-confirm-apply flow with the user kept in the loop;
- **When** an AI auditor or implementer walks spec/22 §97 and seeks the binding contract for the auto-fix protocol — download request, fix-bundle envelope, 4-state outcome enumeration, preflight preconditions, atomic apply, post-apply pointer-file refresh, error envelope,
- **Then** the auditor MUST find the following seven normative sub-contracts pinned in `44-cli-autofix-protocol.md` (delegated by name from this AC, not restated here per Lesson #21 discipline):

  1. **Download request** — `GET <autofixDownloadUrl>` (no rewriting; no query-param mutation); `https`-only (HTTP redirect rejected with `INTERNAL`); SSH-key Lane B auth per spec/22 §31 (link-don't-restate); `Accept: application/vnd.riseup.git-logs.fix+json; v=1`; `Accept-Encoding: gzip`; conditional `If-None-Match: <etag>`; max body 4 MiB compressed / 16 MiB decompressed; 30s connect + 60s body timeout. FORBIDDEN: cross-origin redirects, re-using §43 `Idempotency-Key`, persisting body before validation passes.
  2. **Fix-bundle envelope (closed 7-field set)** — `bundleId` (UUIDv4) · `batchId` (UUIDv4 — MUST equal triggering §43 batch) · `repoIdentityHash` (64-hex sha256) · `targetHead` (40-hex SHA) · `schemaBump` (enum {`patch`,`minor`,`major`}) · `patches` (array of `{path, mode, beforeSha256, afterSha256, unifiedDiff}` — `additionalProperties: false`) · `expiresAt` (RFC 3339 UTC). 17-entry forbidden-field list (mirrors §43 §3 16-entry + `executablePath`).
  3. **4-state outcome (closed set)** — `APPLIED` → `ExitCode.OK`; `REJECTED_USER` → `ExitCode.OK`; `REJECTED_PRECONDITION` → `ExitCode.UserError`; `DEFERRED_NETWORK` → `ExitCode.Internal` (per spec/13 AC-21 link-don't-restate). Every emitted state MUST be uploaded as an §43 frame with `subcommand: "fix"` and `outcome` mapped to `NORMAL`/`WARN`/`ERROR`/`INTERNAL` respectively.
  4. **Preflight preconditions (strict 7-step order)** — (1) `autofixDownloadUrl` freshness · (2) `repoIdentityHash` match · (3) `targetHead` ≡ current HEAD · (4) `expiresAt` future · (5) working tree clean (`git status --porcelain=v2 -z` empty) · (6) per-patch `beforeSha256` match · (7) `schemaBump == "major"` REQUIRES interactive confirm (`--auto-confirm` IGNORED for major; `REJECTED_USER`); first failure short-circuits to `REJECTED_PRECONDITION` + matching `GL-FIX-*` code. Reordering FORBIDDEN. Auto-`git stash` FORBIDDEN.
  5. **Atomic apply discipline** — compute all `(path, afterContent)` tuples in memory FIRST; verify each `afterSha256`; write each via temp-then-rename per spec/13 AC-22 (link-don't-restate) using `<path>.riseup-fix.<bundleId>.tmp` + `fsync` + `rename()`; per-file atomic, not bundle-transactional; mid-bundle rename failure → STOP, emit `DEFERRED_NETWORK` with partially-applied paths, NO automatic rollback. FORBIDDEN: in-place writes, shelling out to `git apply`, auto-mkdir of missing parents, chmod beyond preserving mode.
  6. **Post-apply pointer-file refresh** — on `APPLIED` only: re-derive `currentHead` (belt-and-braces invariant); update `.riseup-git-logs.json` per §41 atomic write; set `uploadedAt` + `expiresAt` + `autofixDownloadUrl: null` (URL consumed, MUST NOT be reused); user MUST manually `git add` + `git commit` (CLI does NOT auto-commit). FORBIDDEN: auto-commit, auto-push, mutating any pointer-file field other than the three listed, running any `git` write command inside the auto-fix flow.
  7. **Error envelope (11-row mapping)** — `GL-FIX-URL-STALE` · `GL-FIX-IDENTITY-MISMATCH` · `GL-FIX-HEAD-DRIFT` · `GL-FIX-EXPIRED` · `GL-FIX-DIRTY-TREE` · `GL-FIX-BEFORE-MISMATCH` · `GL-FIX-AFTER-MISMATCH` · `GL-FIX-PAYLOAD-TOO-LARGE` · `GL-FIX-AUTH-INVALID` · `GL-FIX-SERVER-ERROR` (codes owned by spec/22 §15 — link-don't-restate per Lesson #36).

**AND** the J-series sibling slot §45 ships under **AC-87** (Phase J15 — `45-cli-test-plan.md`, AC-80 row 42 added in this phase);

**AND** any of these patterns is FORBIDDEN: (a) restating any of the 7 sub-contracts inline in §40, §41, §42, §43, or §45 — those files MUST cite §44 by section anchor only; (b) restating spec/13 AC-21 `ExitCode` enum or AC-22 atomic-write discipline inside §44 (Lesson #36 violation); (c) emitting any state outside the 4-state closed set; (d) treating `--auto-confirm` as bypassing `major` schema bumps; (e) auto-committing or auto-pushing applied patches; (f) shelling out to `git apply`;
